## Usage

<DocSection>
  The `Text` component lets you define hierarchy once and reuse it across native screens, settings pages, and longer-form product surfaces.

### Headings

<DocSubsection>
  Use heading variants to create clear section breaks and semantic roles.
  <CompPreview
    code={`<Text variant="h1">Ship native UI faster.</Text>
<Text variant="h2">Overview</Text>
<Text variant="h3">Installation</Text>`}
  >
    <TextHeadingsInlinePreview />
  </CompPreview>
</DocSubsection>

### Body Copy

<DocSubsection>
  Pair `p`, `lead`, `muted`, and `small` for readable layouts with more contrast
  control.
  <CompPreview
    code={`<Text variant="lead">A composable type system for React Native.</Text>
<Text variant="p">
  Build headings, helper text, and long-form copy from one primitive.
</Text>
<Text variant="muted">Updated 2 minutes ago</Text>`}
  >
    <TextBodyInlinePreview />
  </CompPreview>
</DocSubsection>

## API Reference

<DocSection>
  `Text` extends the React Native `Text` API and adds a `variant` prop for
  semantic styling.
  <ApiTable
    rows={[
      {
        prop: "variant",

        default: `"default"`,
        description: "Selects the typography scale and semantic role.",
      },
      {
        prop: "asChild",
        type: "boolean",

      {
